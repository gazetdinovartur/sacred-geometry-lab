## Роли и зоны ответственности

### Product Owner — что строим и зачем

#### Миссия

Sacred Geometry Lab — **лаборатория наблюдения**, где внутренние процессы становятся видимыми через математически точное отображение звука в геометрию. Пользователь *видит форму того, что звучит*, созерцает её и исследует своё состояние — без оценок и диагнозов.

#### Чем проект **является**

- Инструмент **самопознания** (ядро продукта)
- Инструмент визуализации музыки и голоса
- Генеративная художественная лаборатория
- Мастерская и безопасное место для исследования состояний
- Пространство, где узор может читаться как **метафорическая ассоциативная карта** или **буддийская мандала** — без навязывания интерпретации

#### Чем проект **не является**

| Не является | Почему это важно сказать явно |
|-------------|-------------------------------|
| Живым человеком, терапевтом, наставником | Ни один инструмент не заменяет живое человеческое общение |
| Диагностикой, психологической оценкой | Лаборатория не интерпретирует и не ставит диагнозов |
| Редактором SVG / генератором «красивых картинок» / галереей сакральной геометрии | Форма рождается из звука, а не из пресетов |
| Системой распознавания речи и анализа содержания | Слова не хранятся и не анализируются семантически |
| Гарантом «истины о человеке» | Только наблюдаемые акустические проявления → геометрия |

#### Целевая аудитория (приоритет)

1. Автор проекта — личный инструмент исследования
2. Музыканты, вокалисты, исследователи ритма
3. Люди в терапевтическом / созерцательном контексте (как клиент, не как замена специалиста)
4. Художники, преподаватели, любознательные

#### Два режима работы (ключевое продуктовое решение)

| Режим | Название в UI | Суть |
|-------|---------------|------|
| **Live** | «Мандала момента» | Геометрия строится **в реальном времени** по мере звучания. Состояние ↔ форма меняются синхронно. |
| **Process** | «След процесса» | Визуализация **этапов и итога**: слепки (snapshots) по ходу сессии, таймлайн трансформации, финальный композит. Отвечает на «как менялось» — не только «как сейчас». |

Оба режима используют **один и тот же математический движок** — различается только представление (поток vs архив).

### UX Architect — как это ощущается

#### Принципы интерфейса

1. **Минимализм** — мало элементов, много воздуха; внимание на мандале, не на хром
2. **Отзывчивость** — каждое изменение звука немедленно отражается в форме; UI не «думает» дольше 16 ms на кадр визуализации
3. **Мягкость** — тон «Как ты сейчас?», не «Запишите голос для анализа»
4. **Laboratory, not dashboard** — ощущение мастерской, не SaaS-аналитики
5. **Долгое пребывание** — плавные micro-animations, приятная типографика
6. **Светлая и тёмная тема** — переключатель в UI; по умолчанию — системная (`prefers-color-scheme`); обе темы полноценные, не «инверсия как afterthought»

#### Математическая связь звук → образ

**Принцип:** каждый визуальный параметр — **прямая функция** измеряемой величины. Документированная формула. Без ML, без «улучшайзеров», без случайности (кроме seed от hash профиля пользователя для уникальности).

**Базовые маппинги (из файла + расширения):**

| Акустическая величина | Как измеряем | Геометрический параметр |
|-----------------------|--------------|-------------------------|
| RMS (громкость) | AnalyserNode time domain | радиус базовой окружности |
| f₀ (основная частота) | autocorrelation / YIN | количество лучей (квантизация по октавам) |
| Спектр (FFT bins) | AnalyserNode frequency | цвет (hue ← spectral centroid), вложенные кольца (гармоники) |
| Onset detection (удары) | spectral flux peaks | симметрия N (3→△, 4→□, 5→☆, 7→7-луч) |
| Тишина (silence ratio) | RMS < threshold | **покой** — см. SilenceMapper ниже, не «дыры» |
| Длительность ноты | envelope sustain | длина / толщина элементов |
| Pitch contour | f₀ over time | спираль pitch, наклон лучей |
| Overtone richness | harmonic peaks count | число вложенных уровней геометрии |
| Spectral centroid | weighted mean freq | «яркость» — толщина линий, inner glow |
| Spectral flux | frame-to-frame Δ spectrum | скорость морфинга / вращения |
| Zero-crossing rate | crossings / sample | гладкость vs зубчатость контура |
| HNR (harmonic-to-noise) | harmonic / noise energy | резкость vs размытие краёв |
| Formant bands (F1–F3) | LPC / peak picking | внутренний многоугольник ( **без** привязки к словам) |
| Dynamic range | max RMS − min RMS сессии | плотность концентрических колец |
| Inter-onset interval variance | σ of IOI | правильность vs «organic» asymmetry |
| Phase correlation bands | cross-correlation | opacity overlay между слоями |
| Jitter / shimmer | micro-variation f₀ | лёгкая асимметрия (уникальность живого голоса) |

**Уникальность человека:**

При первой сессии (или в кабинете) — **калибровка 10–15 сек**: определяем диапазон f₀, средний тембр (spectral envelope), типичный RMS. Далее все маппинги **нормализуются** относительно этого профиля → мандала «его/её», а не абстрактная.

Формула уникальности (детерминированная):

```
seed = hash(voiceProfileId + sessionTimestamp)
asymmetryOffset = deterministicNoise(seed) * jitterFactor
```

#### Тишина — не negative space

Negative space (вырезать дырки в форме) — **визуальная метафора**, не акустическая реальность. В звуке тишина — это низкая энергия, пауза, фаза покоя; форма при этом **не исчезает**, а меняет динамику.

**Принятое решение — «Rest & Breath»:**

| Параметр при тишине | Формула / поведение |
|---------------------|---------------------|
| Вращение | `rotationSpeed *= (1 - silenceRatio)` — замедление, не стоп |
| Новые элементы | не добавляются; существующая форма **сохраняется** |
| «Дыхание» | тонкое внешнее кольцо, радиус ∝ длительность паузы (`pauseMs → ringRadius`) |
| Накопление | долгая пауза слегка повышает «заряд» — следующий звук ярче (amplitude boost cap) |
| Прозрачность | только у **новых** штрихов ↓, не у всей мандалы |

Альтернативы (можно A/B в dev): **Trace hold** — последняя форма пульсирует с затуханием; **Micro-reveal** — при тишине проявляются мелкие детали от noise floor. Основной режим — Rest & Breath.

**Близость реальностей звук ↔ образ:**

- 60 fps визуализация синхронизирована с audio clock
- Smoothing только экспоненциальный (настраиваемый τ), не «красивости»
- Process-режим: каждый слепок = **снимок feature vector** в момент T, не перерисовка «задним числом»
- Side-by-side debug (dev mode): waveform + spectrum + mandala для верификации

#### Live vs Process — технически

| | Live | Process |
|---|------|---------|
| Loop | rAF + AnalyserNode | тот же extractor, буфер feature frames[] |
| Snapshot trigger | — | каждые N сек OR значимое изменение (Δ centroid > ε) |
| Render | один canvas, continuous | timeline scrub + composite layer |
| Export | текущий кадр | серия PNG / contact sheet / SVG с layers |

#### Производительность

- AudioWorklet для feature extraction (не блокировать main thread)
- Paper.js: переиспользовать paths, не пересоздавать scene каждый frame
- Mobile: fallback упрощённой геометрии (меньше элементов), но **те же формулы**

### Backend — API, auth, хранение

#### Стек

- **Symfony 8**, PHP 8.4+
- **MySQL** — пользователи, узоры (метаданные), OAuth bindings
- **Docker** локально, NetAngels на проде
- Домен: **lab.arturlun.ru** (поддомен arturlun.ru)

#### Принцип: сервер не слышит

В MVP **аудио на сервер не передаётся**. Backend хранит только:

- учётная запись (OAuth)
- **Pattern** — сохранённый узор:
  - `geometryParams` (JSON — числа, не аудио)
  - `featureTimeline` (Process — массив snapshot vectors)
  - `svg` или ссылка на static storage
  - `thumbnail` PNG
  - `voiceProfileHash` (анонимизированный, не reversable в голос)
  - `createdAt`, `title` (опционально, пользовательское)
  - `mode`: live | process

## Дополнительные источники целостного образа

Помимо базовых параметров из Sacred Geometry Lab.txt, для **честного и целостного** выхода предлагается:

1. **Тишина как покой (Rest & Breath)** — паузы замедляют, сохраняют форму, рисуют «дыхательное» кольцо; молчание между словами из Анонса — first-class параметр
2. **Dual-source geometry (диалог)** — два голоса → два полушария / interlocking структуры; overlap = зона контакта (интенсивность, не «совместимость»)
3. **Temporal arc (Process)** — не один кадр, а **история трансформации** — как в файле: «несколько слепков из процесса»
4. **Rhythmic layer stacking (полиритм)** — несколько IOI → наложение геометрий с разной симметрией
5. **Formant geometry (для голоса)** — формантные полосы → внутренний многоугольник; отражает *как* звучит голос, не *что* сказано
6. **Breath detection** — низкочастотные всплески + паузы → «пульс» мандалы, цикл вдох-выдох
7. **Session envelope** — динамический диапазон всей сессии → от центра к периферии, «путь» от начала к концу
8. **Composite mandala (Process итог)** — финальный кадр = superposition слепков с весами по времени; **след прожитого**, не усреднение

Все вос восемь — **измеримые величины**, не интерпретации.
